# 삼평동 책상 전략 연구소 (sdl) E2E 테스트 명세서

## 1. 회원가입 (Sign Up)

### TC-01-01: 필수 입력값 누락 시 회원가입 차단 및 경고 표시
* **테스트 목적**: 필수 값(실명, 부서)이 누락되었을 때 폼 전송이 차단되는지 확인합니다.
* **수행 단계**:
  1. `/signup` 페이지로 이동합니다.
  2. 닉네임 필드에 `testuser`를 입력합니다.
  3. 실명 필드와 부서 필드를 빈칸으로 둡니다.
  4. 가상 키패드를 활성화하여 비밀번호 `1234`를 입력합니다.
  5. 회원가입 버튼을 클릭합니다.
* **기대 결과**:
  * 폼이 제출되지 않고, "이름과 부서는 필수 입력 항목입니다."라는 안내 팝업 또는 경고 메시지가 노출됩니다.

### TC-01-02: 중복 닉네임 가입 차단
* **테스트 목적**: 이미 가입된 닉네임으로 가입 시도 시 가입이 거부되는지 확인합니다.
* **수행 단계**:
  1. `/signup` 페이지로 이동합니다.
  2. 이미 등록된 닉네임인 `already_taken`을 입력합니다.
  3. 실명 `김철수`, 부서 `개발팀`을 입력합니다.
  4. 가상 키패드로 비밀번호 `1111`을 입력합니다.
  5. 회원가입 버튼을 클릭합니다.
* **기대 결과**:
  * "이미 사용 중인 닉네임입니다."라는 중복 오류 경고가 표시되며 화면이 가입 페이지에 머무릅니다.

### TC-01-03: 가상 키패드 정상 입력 및 마스킹 처리
* **테스트 목적**: 물리 키보드 입력을 제한하고 가상 키패드 클릭으로만 비밀번호가 안전하게 마스킹 입력되는지 검증합니다.
* **수행 단계**:
  1. `/signup` 페이지로 이동합니다.
  2. 비밀번호 입력 필드를 클릭합니다.
  3. 물리 키보드로 숫자 `9`를 쳐보고 입력이 무시되는지 확인합니다.
  4. 노출된 가상 키패드 버튼 중 `5`, `6`, `7`, `8`을 순서대로 클릭합니다.
* **기대 결과**:
  * 물리 키보드 입력은 처리되지 않아야 합니다.
  * 비밀번호 입력창에 `••••` 또는 `****`로 마스킹된 4글자가 표시되어야 합니다.

### TC-01-04: 회원가입 성공 및 피드로 리다이렉션
* **테스트 목적**: 올바른 입력 조건에서 가입이 성공하고 로그인 세션 상태로 피드 화면으로 이동하는지 검증합니다.
* **수행 단계**:
  1. `/signup` 페이지로 이동합니다.
  2. 중복되지 않는 고유 닉네임, 실명, 부서를 입력합니다.
  3. 가상 키패드를 통해 비밀번호 4자리를 입력합니다.
  4. 가입 버튼을 누릅니다.
* **기대 결과**:
  * 가입 처리가 성공하고 자동으로 로그인 세션이 생성됩니다.
  * 브라우저 URL이 `/feed`로 변경됩니다.

## 2. 로그인 및 세션 관리 (Auth)

### TC-02-01: 로그인 정보 인증 실패 검증
* **테스트 목적**: 가입되지 않은 정보나 잘못된 비밀번호 입력 시 로그인이 차단되는지 검증합니다.
* **수행 단계**:
  1. `/login` 페이지로 이동합니다.
  2. 존재하지 않는 닉네임 `nobody`를 입력하고 패스워드 `9999`를 가상 키패드로 입력해 제출합니다.
  3. 가입된 닉네임 `testuser`를 입력하고 잘못된 패스워드 `0000`을 가상 키패드로 입력해 제출합니다.
* **기대 결과**:
  * 두 시도 모두 "닉네임 또는 비밀번호가 일치하지 않습니다." 에러가 표시됩니다.

### TC-02-02: 정상 로그인 및 세션 유지
* **테스트 목적**: 정상 정보를 입력하여 로그인 상태가 유지되는지 검증합니다.
* **수행 단계**:
  1. `/login` 페이지로 이동합니다.
  2. 가입된 닉네임과 올바른 비밀번호 4자리를 가상 키패드로 입력합니다.
  3. 로그인 버튼을 클릭합니다.
* **기대 결과**:
  * URL이 `/feed`로 성공적으로 이동합니다.
  * 페이지를 새로고침(`F5`)해도 로그인 세션이 풀리지 않고 계속 `/feed`를 유지합니다.

### TC-02-03: 로그아웃 처리
* **테스트 목적**: 로그아웃 시 로그인 세션이 완전 파괴되고 비인증 상태로 복귀하는지 검증합니다.
* **수행 단계**:
  1. 로그인 상태에서 네비게이션 바의 로그아웃 버튼을 클릭합니다.
* **기대 결과**:
  * 로그아웃 즉시 로그인 화면 `/login`으로 리다이렉션됩니다.
  * 브라우저 뒤로 가기를 눌러도 피드 화면(`/feed`)으로 돌아가지 않고 다시 로그인 화면으로 튕깁니다.

## 3. 회원 정보 관리 & 명부 (Profiles & Directory)

### TC-03-01: 프로필 정보 편집 반영
* **테스트 목적**: 닉네임, 실명, 부서, 자기소개 문구가 올바르게 수정 및 반영되는지 검증합니다.
* **수행 단계**:
  1. `/profile/edit` 페이지로 접속합니다.
  2. 실명을 `수정실명`, 부서를 `수정부서`로 변경하고, 자기소개 필드에 `새로운 소개글`을 기재합니다.
  3. 수정 완료 버튼을 누릅니다.
* **기대 결과**:
  * 수정 완료 팝업이 노출되며, 명부나 프로필 화면에서 수정된 정보가 표시됩니다.

### TC-03-02: 프로필 사진 업로드 및 반영
* **테스트 목적**: Supabase Storage 연동을 통한 프로필 이미지 등록 프로세스를 테스트합니다.
* **수행 단계**:
  1. 프로필 수정 페이지에서 프로필 이미지 변경 영역을 클릭합니다.
  2. 샘플 이미지 파일(`test.png`)을 선택하여 업로드합니다.
  3. 저장 버튼을 누릅니다.
* **기대 결과**:
  * 업로드 완료 후 프로필 이미지 소스 URL이 Supabase Storage 경로로 변경되어 정상 노출됩니다.

### TC-03-03: 회원 탈퇴 플로우
* **테스트 목적**: 회원 탈퇴 시 경고 알림 및 데이터 파기 절차가 정상 작동하는지 확인합니다.
* **수행 단계**:
  1. 설정 또는 프로필 편집 하단의 "회원 탈퇴" 버튼을 클릭합니다.
  2. "정말로 탈퇴하시겠습니까?" 컨펌 모달에서 "확인"을 클릭합니다.
* **기대 결과**:
  * 사용자 계정이 삭제 처리되고 세션이 만료되어 비인증 첫 화면으로 리다이렉션됩니다.
  * 기존 닉네임으로 다시 로그인을 시도했을 때 로그인에 실패해야 합니다.

### TC-03-04: 회원 명부 조회 및 실시간 검색 필터링
* **테스트 목적**: 동아리 회원 명부 조회 기능과 검색창 입력을 통한 목록 필터링을 검증합니다.
* **수행 단계**:
  1. `/directory` 페이지로 진입합니다.
  2. 전체 회원 리스트가 카드나 목록 형태로 노출되는지 확인합니다.
  3. 검색창에 특정 부서명(예: `기획팀`)을 입력합니다.
* **기대 결과**:
  * 기획팀 부서에 속한 회원 정보만 목록에 남고 다른 부서 회원들은 화면에서 실시간 필터링 제거되어야 합니다.

## 4. 인스타그램 스타일 자유 게시판 (Feeds)

### TC-04-01: 새로운 피드 작성 (텍스트 & 이미지)
* **테스트 목적**: 텍스트 내용과 이미지를 업로드하여 피드를 작성하고 최상단 배치 동작을 검증합니다.
* **수행 단계**:
  1. 피드 페이지에서 "새 글 쓰기" 버튼을 누릅니다.
  2. 본문에 `테스트 게시물 내용입니다. #보드게임`을 입력합니다.
  3. 이미지 업로드 버튼을 눌러 사진 파일을 첨부합니다.
  4. 등록 버튼을 누릅니다.
* **기대 결과**:
  * 피드 리스트의 맨 처음에 방금 등록한 본문과 업로드된 이미지가 정상 노출됩니다.

### TC-04-02: 피드 작성 권한 및 수정/삭제 검증
* **테스트 목적**: 타인의 피드는 삭제할 수 없으며 본인이 쓴 글만 CRUD 제어가 가능한지 검증합니다.
* **수행 단계**:
  1. 로그인 상태에서 타인의 피드를 찾아서 더보기(메뉴) 버튼을 찾아봅니다.
  2. 내가 작성한 피드를 찾아 더보기 버튼을 누르고 "삭제"를 클릭합니다.
* **기대 결과**:
  * 타인의 피드 영역에는 수정 및 삭제 메뉴 아이콘 자체가 보이지 않습니다.
  * 내가 쓴 피드는 삭제 성공 후 피드 목록 화면에서 즉각 제거됩니다.

### TC-04-03: 무한 스크롤(Infinite Scroll) 렌더링
* **테스트 목적**: 페이지 스크롤 동작 시 다음 데이터 페이지의 피드들이 중단 없이 덧붙는지 검증합니다.
* **수행 단계**:
  1. `/feed` 페이지에 진입하여 초기에 로드된 피드 개수(예: 5개)를 확인합니다.
  2. 스크롤바를 윈도우 최하단부로 스크롤합니다.
* **기대 결과**:
  * 로딩 스피너 혹은 스켈레톤이 나타났다 사라진 뒤, 추가 피드가 로드되어 목록 뒤에 순서대로 이어집니다.

### TC-04-04: 해시태그 기반 피드 모아보기
* **테스트 목적**: 피드 본문 내 해시태그 링크 클릭 시 필터링 뷰로 안전하게 진입하는지 확인합니다.
* **수행 단계**:
  1. 피드 중 `#보드게임` 태그가 적힌 피드를 찾습니다.
  2. `#보드게임` 파란색 링크 텍스트를 클릭합니다.
* **기대 결과**:
  * URL이 `/feed?tag=보드게임` 형태로 이동하거나 필터링 모드로 전환됩니다.
  * 리스트에 `#보드게임` 해시태그를 포함한 피드들만 노출됩니다.

### TC-04-05: 좋아요 토글 기능
* **테스트 목적**: 좋아요 버튼 클릭 시 카운팅 증감 및 색상 토글이 매끄럽게 연동되는지 검증합니다.
* **수행 단계**:
  1. 임의의 피드 하단에 있는 빈 하트(좋아요) 아이콘을 클릭합니다.
  2. 카운트 숫자를 확인한 후 다시 채워진 하트 아이콘을 클릭합니다.
* **기대 결과**:
  * 첫 클릭 시 하트 아이콘이 빨간색으로 활성화되며 좋아요 숫자가 1 증가합니다.
  * 재클릭 시 하트 아이콘이 다시 빈 모양으로 변경되며 좋아요 숫자가 1 감소합니다.

## 5. 댓글 시스템 (Comments)

### TC-05-01: 실시간 댓글 등록
* **테스트 목적**: 피드 밑 댓글 창에 텍스트 입력 후 등록 시 화면 리로드 없이 화면에 즉시 갱신되는지 확인합니다.
* **수행 단계**:
  1. 임의의 피드에서 댓글 입력 칸을 활성화합니다.
  2. `이 보드게임 재밌어 보이네요!` 라고 쓰고 등록(전송)을 누릅니다.
* **기대 결과**:
  * 전체 페이지 새로고침 없이 작성한 닉네임과 함께 해당 댓글이 즉시 리스트 최하단에 렌더링됩니다.

### TC-05-02: 댓글 삭제 동작 및 권한
* **테스트 목적**: 내 댓글만 삭제 버튼이 노출되며 정상 삭제가 가능한지 확인합니다.
* **수행 단계**:
  1. 타인이 작성한 댓글 영역을 살펴봅니다.
  2. 내가 작성한 댓글 옆에 노출된 삭제(휴지통) 버튼을 누릅니다.
* **기대 결과**:
  * 타인의 댓글에는 삭제 버튼이 보이지 않습니다.
  * 내 댓글은 삭제 확인 처리 즉시 목록에서 제거됩니다.

## 6. 공지 사항 (Announcements)

### TC-06-01: 일반 회원 권한 제한
* **테스트 목적**: 일반 사용자 계정은 공지를 쓰거나 수정할 수 없음을 검증합니다.
* **수행 단계**:
  1. 일반 회원으로 로그인하여 `/announcements` 페이지로 이동합니다.
  2. 화면 내에 "공지 등록" 또는 "글쓰기" 버튼이 존재하는지 확인합니다.
* **기대 결과**:
  * 글쓰기/등록 관련 버튼 및 인터페이스가 전혀 보이지 않아야 합니다.

### TC-06-02: 관리자 권한의 공지 CRUD 제어
* **테스트 목적**: 관리자 권한을 가진 계정으로 접속했을 때는 공지 글쓰기 및 수정/삭제가 원활히 동작하는지 확인합니다.
* **수행 단계**:
  1. `role = 'admin'` 상태인 관리자 계정으로 로그인합니다.
  2. `/announcements` 페이지로 진입하여 "공지 등록"을 클릭합니다.
  3. 제목과 내용을 적고 저장한 후, 수정 및 삭제 프로세스를 거칩니다.
* **기대 결과**:
  * 공지사항 작성, 수정, 삭제 일련의 과정이 정상적으로 수행되어 목록에 반영됩니다.
