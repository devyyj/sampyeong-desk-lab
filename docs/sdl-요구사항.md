# 삼평 데스크 랩 (Sampyeong Desk Lab - SDL) 웹 서비스 요구사항

## 1. 서비스 개요
- **서비스명**: 삼평동 책상 전략 연구소 (sdl)
- **목적**: 사내 보드게임 동호회 회원들 간의 소통 및 보드게임 인프라 관리를 위한 커뮤니티 웹 서비스
- **타겟 유저**: 동호회 가입 회원 및 가입을 고려하는 예비 회원

## 2. 주요 메뉴 및 기능 요구사항

### 2.1. 공통 정책 및 레이아웃
- **랜딩 페이지**: 서비스 첫 접속 시 기본 화면은 **자유게시판**으로 설정
- **네비게이션 바**: 화면 상단에 네비게이션 바(GNB)가 **항상 고정 노출**됨
- **권한 및 접근성**: 비회원(로그인하지 않은 사용자)도 **모든 메뉴의 조회(읽기)는 가능**하며, 게시글 작성 등 데이터 변경 작업은 회원만 가능

### 2.2. 자유게시판
- **UI/UX**: 인스타그램 피드 형식 (사진과 글이 메인 화면에 바로 노출)
- **목록 조회**: 무한 스크롤(Infinite Scroll) 방식을 적용하여 끊김 없는 탐색 경험 제공
- **게시글 작성**: 사진 첨부 및 텍스트 작성 기능
- **상호작용 (확장 가능성)**: 댓글 작성, 좋아요 기능
- **관리**: 본인이 작성한 게시글 수정 및 삭제 기능

### 2.3. 회원 목록
- **정보 조회**: 가입된 동호회 회원들의 프로필 정보를 리스트 형태로 제공
- **표시 정보**: 프로필 사진, 아이디(또는 이름), 소속 부서(팀), 한 줄 자기소개
- **검색/필터 (확장 가능성)**: 부서별 필터링 또는 이름 검색

### 2.4. 보드게임 보유 현황
- **목록 조회**: 동호회에서 보유 중인 보드게임 리스트 확인
- **상세 정보**: 게임명, 추천 인원수, 플레이 시간, 썸네일 이미지 등
- **상태/대여 관리 (확장 가능성)**: 현재 플레이 중이거나 대여 중인지 상태 표시
- **관리 기능**: 새로운 보드게임 추가 및 기존 정보 수정/삭제 (관리자 권한 고려)

### 2.5. 사용자 계정 및 마이페이지
- **회원가입 및 로그인**:
  - 심리적 진입 장벽을 낮추기 위해 **비밀번호는 4자리 숫자(PIN)** 로 제한
  - 가입 시 입력 항목: 아이디, 4자리 비밀번호, 소속 부서(팀)
  - 로그인/로그아웃 상태에 따라 상단 네비게이션 바(GNB)의 메뉴 구성 변경
- **내 정보 (마이페이지)**:
  - 프로필 사진 등록 및 변경
  - 자기소개 텍스트 수정
  - 비밀번호 변경
  - 회원 탈퇴

## 3. 데이터베이스 및 인프라 (Supabase)
- **BaaS**: Supabase 활용
- **인증 (Auth)**: 4자리 PIN 번호 인증을 위한 Custom Auth 로직 또는 별도 테이블 인증 방식 고려
- **데이터베이스 (DB)**: Users, Posts, BoardGames 등 주요 테이블 설계 필요
- **스토리지 (Storage)**: 프로필 사진, 자유게시판 피드 이미지, 보드게임 썸네일 저장

## 4. UI/UX 디자인 원칙
- **직관성과 심플함**: 복잡성을 배제하고 누구나 쉽게 사용할 수 있는 직관적인 UI
- **반응형 웹 (모바일 최우선)**: 모바일 환경에서 피드 스크롤이나 보드게임 현황을 보기 편하도록 최적화
