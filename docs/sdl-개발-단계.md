# 삼평 데스크 랩 (SDL) 개발 단계 계획서

본 문서는 `sdl-요구사항.md`에 정의된 요구사항을 바탕으로 전체 프로젝트 개발 단계를 논리적으로 나눈 계획서입니다.

---

## 1단계 (Phase 1): 기획 및 인프라 설계
- **요구사항 리뷰**: 구체적인 정책 및 기능 범위 확정
- **UI/UX 와이어프레임**: 모바일 최우선(Mobile-First) 디자인을 반영한 각 페이지 화면 설계
- **데이터베이스 모델링**:
  - Supabase 테이블 설계 (`Users`, `Posts`, `BoardGames` 등)
  - 테이블 관계(Relations) 및 권한(RLS) 정책 수립
- **스토리지 세팅**: Supabase Storage 버킷 생성 (프로필 사진용, 게시글 이미지용, 보드게임 썸네일용)

## 2단계 (Phase 2): 프로젝트 초기 세팅
- **프레임워크 초기화**: Next.js 프로젝트 생성 및 설정 (App Router 적용 등)
- **코드 컨벤션 세팅**: ESLint, Prettier 등 린트 툴 구성
- **디자인 시스템 및 스타일링 셋업**: 반응형 및 기본 테마 컬러 지정, CSS(또는 Tailwind CSS) 설정
- **Supabase 연동**: 환경변수 설정 및 Supabase 클라이언트 SDK 초기화

## 3단계 (Phase 3): 공통 레이아웃 및 사용자 인증
- **공통 컴포넌트 개발**:
  - 항상 고정되는 **상단 네비게이션 바(GNB)** 컴포넌트 개발
- **인증 시스템(Auth) 구현**:
  - 4자리 PIN 번호를 활용한 커스텀 회원가입 및 로그인 로직 구현
  - 로그인/로그아웃 상태에 따른 GNB 메뉴 구성 동적 변경 처리
- **라우팅 및 접근 권한 제어**:
  - 메인(랜딩) 페이지를 **자유게시판**으로 연결
  - 비회원(로그인하지 않은 상태)도 조회 가능한 페이지 접근 허용 로직 처리
  - 데이터 변경(글쓰기, 수정) 페이지에 대한 라우팅 가드(Middleware) 설정

## 4단계 (Phase 4): 핵심 기능 - 자유게시판 (Landing Page)
- **UI 구현**: 인스타그램 스타일의 피드 화면 구성
- **목록 조회**: 게시글 목록 조회 및 **무한 스크롤(Infinite Scroll)** 적용
- **게시글 작성/관리**:
  - 텍스트 입력 및 이미지 업로드(Supabase Storage) 기능 연동
  - 본인이 작성한 게시글 수정 및 삭제 기능
- **상호작용(선택/확장)**: 댓글 작성 및 좋아요 기능 추가

## 5단계 (Phase 5): 마이페이지 및 회원 목록
- **마이페이지**:
  - 사용자 프로필 사진 업로드 및 정보 불러오기
  - 자기소개 텍스트 수정 및 4자리 비밀번호 변경 기능
  - 회원 탈퇴 로직 구현
- **회원 목록**:
  - 전체 가입자 정보를 리스트업하는 UI 구성
  - 프로필 사진, 아이디, 소속 부서, 자기소개 표시

## 6단계 (Phase 6): 보드게임 보유 현황
- **목록 조회**: 동호회에서 보유 중인 보드게임 리스트 뷰(Grid/List) 구현
- **상세 정보 컴포넌트**: 추천 인원, 플레이 시간, 게임 이미지 등 노출
- **상태 관리 및 등록 (관리자용)**:
  - 새로운 보드게임 추가 및 기존 정보 수정/삭제 기능
  - 현재 플레이 중인지, 대여 중인지 상태를 관리할 수 있는 토글 또는 선택 기능 추가

## 7단계 (Phase 7): 테스트, QA 및 배포
- **통합 테스트 및 QA**: 데스크톱 및 모바일 기기에서의 반응형 디자인 및 브라우저 호환성 검증
- **오류 수정 (Bug Fix)**: 발견된 이슈 해결 및 최적화
- **프로덕션 배포**: Vercel 등 호스팅 플랫폼을 통한 서비스 라이브 배포 및 도메인 연결
