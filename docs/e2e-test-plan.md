# SDL(Sampyeong Desk Lab) E2E 테스트 계획서

본 문서는 플레이라이트(Playwright)를 활용한 삼평 데스크 랩 웹 서비스의 E2E(End-to-End) 테스트 시나리오와 절차를 정의합니다.

## 1. 테스트 목적
웹 애플리케이션의 주요 흐름(회원가입, 로그인, 게시글 작성, 보드게임 등록, 프로필 수정 등)이 사용자의 관점에서 정상적으로 동작하는지 종단간(End-to-End) 환경에서 검증합니다.

## 2. 테스트 환경 세팅
- **테스트 도구**: Playwright (`@playwright/test`)
- **실행 환경**: 로컬 개발 서버 (`next dev` on port 3000)
- **테스트 대상 브라우저**: Chromium

## 3. 테스트 시나리오

테스트의 안정성과 컨텍스트 유지를 위해 아래의 모든 시나리오는 단일 사용자(User Journey) 흐름으로 연속하여 실행됩니다.

### 시나리오 1: 사용자 인증 흐름 (회원가입 및 세션 확인)
1. 사용자가 `/signup` 페이지에 접속한다.
2. `아이디`, `부서`, `비밀번호(4자리)`를 입력하고 가입을 진행한다.
3. 가입 성공 후, 메인 페이지(`/`)로 리다이렉트 되며 GNB에 `로그아웃` 버튼이 표시되는지 확인한다.

### 시나리오 2: 보드게임 등록 및 조회
1. 로그인된 사용자가 `/boardgames/add` 페이지에 접속한다.
2. `게임명`, `추천 인원수`, `플레이 시간` 등의 정보를 입력하고 등록한다.
3. 등록 완료 후, `/boardgames` 페이지로 리다이렉트 되며 등록한 보드게임이 목록에 표시되는지 확인한다.

### 시나리오 3: 회원 목록 조회
1. 사용자가 `/members` 페이지에 접속한다.
2. 방금 가입한 사용자의 아이디와 부서가 리스트에 노출되는지 확인한다.

### 시나리오 4: 마이페이지 및 프로필 수정 후 로그아웃
1. 사용자가 `/mypage` 페이지에 접속한다.
2. `자기소개` 항목에 새로운 텍스트를 입력하고 저장한다.
3. '정보가 성공적으로 수정되었습니다.' 메시지가 표시되는지 확인하고 데이터가 유지되는지 확인한다.
4. 네비게이션 바에서 `로그아웃` 버튼을 클릭하여 로그아웃한다.
5. `/login` 페이지로 정상 리다이렉트 되는지 확인한다.

## 4. 테스트 실행 방법
1. 로컬 개발 서버 실행: `npm run dev`
2. Playwright 테스트 실행: `npx playwright test`
3. 테스트 결과 리포트 확인.
