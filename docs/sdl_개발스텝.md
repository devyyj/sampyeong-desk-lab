# 삼평동 책상 전략 연구소 (sdl) 개발 단계 (Implementation Steps)

본 프로젝트는 **TDD (테스트 주도 개발)**를 원칙으로 하여 진행됩니다. 각 개발 단계에서는 테스트 코드를 먼저 작성한 후, 구현을 완성하고 리팩터링하는 방식으로 진행합니다.

---

## 단계 1: 환경 구성 및 기본 설정 (Environment Setup)
1. **프로젝트 초기화**
   * Next.js 프로젝트 생성 및 Supabase 클라이언트 라이브러리 설치.
2. **TDD 테스트 프레임워크 설정**
   * Jest, React Testing Library, MSW(Mock Service Worker) 등 테스트 도구 설정.
   * `__tests__` 폴더 구조 및 기초 단위 테스트 검증.
3. **Vanilla CSS / TailwindCSS 테마 설정**
   * 다크모드 대응 컬러 셋, 인스타그램 스타일의 모던한 컴포넌트 디자인 토큰 정의.

---

## 단계 2: Supabase 데이터베이스 및 보안(RLS) 구축
1. **마이그레이션 스크립트 작성**
   * `profiles`, `posts`, `comments`, `likes`, `tags`, `post_tags` 테이블 정의.
2. **RLS (Row Level Security) 정책 정의 및 적용**
   * 각 테이블별 SELECT, INSERT, UPDATE, DELETE 정책 작성.
   * 관리자(Admin) 권한 구분을 위한 정책 정의.
3. **데이터베이스 트리거 및 헬퍼 함수 작성**
   * 신규 Auth 사용자 가입 시 `profiles` 테이블에 자동 연동되는 트리거 작성.

---

## 단계 3: 회원 인증 및 숫자 가상 키패드 구현 (TDD)
1. **테스트 케이스 작성 (`auth.test.ts`, `keypad.test.tsx`)**
   * 가상 키패드 숫자 입력 및 백스페이스 클릭 시 값 변경 테스트.
   * 닉네임, 실명, 부서 유효성 및 4자리 숫자 패스워드 패턴 검증 테스트.
   * Supabase Auth 연동 로그인/회원가입 기능 Mocking 테스트.
2. **기능 구현**
   * 가상 숫자 키패드 컴포넌트 개발.
   * 회원가입 폼 및 로그인 폼 연동.
   * 닉네임 중복 검사 로직 구현.

---

## 단계 4: 프로필 관리 및 회원 명부 구현 (TDD)
1. **테스트 케이스 작성 (`profile.test.tsx`, `directory.test.tsx`)**
   * 프로필 수정 폼 입력값 유효성 검사 및 Supabase Storage 업로드 API 호출 Mocking.
   * 회원 탈퇴 처리 플로우 테스트.
   * 회원 명부 리스트 조회, 검색 및 정렬 필터 테스트.
2. **기능 구현**
   * 프로필 조회 및 편집 페이지 구현 (자기소개 작성 및 프로필 이미지 업로드 기능 포함).
   * 회원 명부 조회 페이지 구현 (부서별/이름별 검색 필터 추가).

---

## 단계 5: 인스타그램 스타일 자유 게시판 및 태그 기능 구현 (TDD)
1. **테스트 케이스 작성 (`feed.test.tsx`, `hashtag.test.ts`)**
   * 무한 스크롤 트리거 시 추가 게시글 호출 테스트.
   * 해시태그 파싱 함수 유틸리티 테스트 (`#보드게임` -> 태그 추출).
   * 게시글/댓글 생성, 수정, 삭제, 좋아요 토글 동작 Mocking 테스트.
2. **기능 구현**
   * 피드형 카드 레이아웃 컴포넌트 및 무한 스크롤(Intersection Observer 활용) 구현.
   * 게시글 CRUD 및 댓글 작성 컴포넌트 구현.
   * 좋아요(Like) 기능 연동.
   * 본문 텍스트 내 해시태그 파싱 및 태그별 모아보기 뷰(View) 구현.

---

## 단계 6: 관리자 공지사항 구현 (TDD)
1. **테스트 케이스 작성 (`announcements.test.tsx`)**
   * 관리자(Admin) 계정인 경우에만 공지사항 쓰기/수정/삭제 버튼이 렌더링되는지 테스트.
   * 일반 회원 계정인 경우 읽기 및 댓글 작성만 가능한지 테스트.
2. **기능 구현**
   * 공지사항 전용 페이지 또는 피드 상단 고정 노출 컴포넌트 구현.
   * 관리자 권한 여부에 따른 분기 처리 및 UI 노출 제어.

---

## 단계 7: 통합 테스트 및 품질 보증 (QA & Polish)
1. **전체 통합 시나리오 테스트**
   * 회원 가입 ➡️ 로그인 ➡️ 피드 작성 ➡️ 댓글 및 좋아요 ➡️ 로그아웃으로 이어지는 전체 흐름 검증.
2. **디자인 마이크로 인터랙션 디테일링**
   * 피드 로딩 시 스켈레톤 UI 적용, 버튼 호버 효과, 좋아요 클릭 애니메이션 보완.
3. **보안 검토**
   * Supabase RLS 정책 재검토 및 SQL Injection 예방 조치 확인.
